Replace the compileWorkflow stub in compiler.ts with the real implementation. The function signature stays the same: `compileWorkflow(spec: LogicSpec, context: WorkflowContext): CompiledWorkflow`.

Implementation steps:

1. **Handle empty steps:** If `!spec.steps` or `Object.keys(spec.steps).length === 0`, return early with `{ steps: [], dagLevels: [], globalQualityGates: [], fallbackPolicy: spec.fallback ?? null, metadata: { name: spec.name, totalSteps: 0, totalLevels: 0 } }`.

2. **Resolve DAG:** Call `resolve(spec.steps)`. If `!dagResult.ok`, throw `new CompilerError("Workflow DAG resolution failed: " + dagResult.errors.map(e => e.message).join("; "))`.

3. **Pre-compile steps in DAG order:** Iterate `dagResult.order` (flat topological order). For each stepName, build an `ExecutionContext` from the WorkflowContext: `{ currentStep: stepName, previousOutputs: context.previousOutputs, input: context.input, attemptNumber: context.attemptNumber, branchReason: context.branchReason, previousFailureReason: context.previousFailureReason }`. Call `compileStep(spec, stepName, ctx)` and collect into a `CompiledStep[]` array.

4. **Compile global quality gates:** If `spec.quality_gates?.pre_output` exists, map each gate to a `QualityGateValidator` using `compileGateValidator(gate.check, gate.message)`. Note: `compileGateValidator` is currently a module-private function — it is already in scope since compileWorkflow is in the same file. No export needed.

5. **Return CompiledWorkflow:**
```typescript
{
  steps: compiledSteps,
  dagLevels: dagResult.levels,
  globalQualityGates: globalGates,
  fallbackPolicy: spec.fallback ?? null,
  metadata: {
    name: spec.name,
    totalSteps: Object.keys(spec.steps).length,
    totalLevels: dagResult.levels.length,
  },
}
```

Run all tests: `npx vitest run packages/core/compiler.test.ts`. All tests MUST PASS. Commit with message: `feat(15-01): implement compileWorkflow with DAG ordering and pre-compiled steps`.
