Create `packages/core/expression.ts` with the following implementation:

**ExpressionError class:**
- Extends Error with `position?: number` field
- Export it

**TokenType enum** (as const enum or regular enum):
Number, String, Boolean, Null, Identifier, Dot, EqualEqual, BangEqual, Less, Greater, LessEqual, GreaterEqual, AmpAmp, PipePipe, Bang, Question, Colon, LeftParen, RightParen, Comma, EOF

**Token interface:** `{ type: TokenType; value: string; position: number }`

**ASTNode discriminated union:**
- Literal: `{ type: "Literal"; value: unknown }`
- Identifier: `{ type: "Identifier"; name: string }`
- MemberExpression: `{ type: "MemberExpression"; object: ASTNode; property: string }`
- BinaryExpression: `{ type: "BinaryExpression"; operator: string; left: ASTNode; right: ASTNode }`
- UnaryExpression: `{ type: "UnaryExpression"; operator: string; operand: ASTNode }`
- ConditionalExpression: `{ type: "ConditionalExpression"; test: ASTNode; consequent: ASTNode; alternate: ASTNode }`
- CallExpression: `{ type: "CallExpression"; callee: ASTNode; property: string; args: ASTNode[] }`

**tokenize(input: string): Token[]** -- Follow the lexer loop from research exactly:
- Skip whitespace
- Match two-char operators first (==, !=, <=, >=, &&, ||)
- Match single-char operators (., <, >, !, ?, :, (, ), ,)
- Match numbers (digits and decimal point)
- Match strings (single and double quoted)
- Match identifiers/keywords (true/false/null become Boolean/Null tokens)
- Throw ExpressionError on unexpected characters
- Append EOF token

**Precedence enum:**
None=0, Ternary=1, Or=2, And=3, Equality=4, Comparison=5, Unary=6, Call=7, Member=8

**parse(tokens: Token[]): ASTNode** -- Pratt parser:
- Maintain a position cursor into the tokens array
- `advance()`, `peek()`, `expect()` helpers
- `parseExpression(precedence)` core loop:
  - Get prefix handler for current token (literals, identifiers, unary !, grouping parens)
  - Loop: while precedence < getPrecedence(peek()), get infix handler (binary ops, dot, ternary, call parens)
- Prefix rules: Number/String/Boolean/Null -> Literal node, Identifier -> Identifier node, Bang -> UnaryExpression, LeftParen -> grouped expression
- Infix rules:
  - Binary operators (==, !=, <, >, <=, >=, &&, ||) -> BinaryExpression at correct precedence
  - Dot -> if next is Identifier followed by LeftParen: parse CallExpression (read comma-separated args until RightParen). Otherwise: MemberExpression.
  - Question -> parse ConditionalExpression (expect Colon between consequent and alternate)
- Export `tokenize` and `parse`

Do NOT use eval() or Function constructor anywhere. Set a max AST depth of 50 to prevent stack overflow.
