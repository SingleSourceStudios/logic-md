<feature>
  <name>compileStep core: strategy preamble + step instructions + metadata</name>
  <files>packages/core/compiler.ts, packages/core/compiler.test.ts</files>
  <behavior>
    Given a LogicSpec with reasoning strategy "react", max_iterations 12, and a step "gather_sources" with description and instructions:

    compileStep(spec, "gather_sources", context) should return CompiledStep where:

    1. systemPromptSegment contains:
       - "## Reasoning Strategy" header followed by "You are using react reasoning." and "Max iterations: 12"
       - Temperature and thinking_budget if present in spec.reasoning
       - "## Current Step: gather_sources" header followed by step.description then step.instructions
    2. metadata.stepName === "gather_sources"
    3. metadata.dagLevel === 0 (gather_sources has no dependencies, so DAG level 0)
    4. metadata.totalSteps === number of steps in spec
    5. metadata.attemptNumber === context.attemptNumber
    6. metadata.branchTaken === context.branchReason (null for Phase 11)
    7. outputSchema === step.output_schema or null
    8. qualityGates === [] (empty array, Phase 13)
    9. retryPolicy === null (handled in Plan 02)
    10. selfReflection === null (Phase 13)

    Error cases:
    - compileStep(spec, "nonexistent", ctx) throws CompilerError
    - compileStep(spec with no steps, "any", ctx) throws CompilerError
    - compileStep(spec with no reasoning, "step", ctx) still works (no strategy section)

    Cases: 
    - gather_sources (root step, dagLevel 0) -> strategy preamble + instructions
    - evaluate_credibility (dagLevel 1) -> correct dagLevel in metadata
    - step with no instructions -> only description in prompt
    - step with no description -> only instructions in prompt
    - spec with no reasoning block -> no strategy preamble section
  </behavior>
  <implementation>
    In compiler.ts, replace the compileStep stub:

    1. Import `resolve` from `./dag.js`
    2. Add internal helper `formatStrategyPreamble(reasoning: Reasoning): string` that formats:
       ```
       ## Reasoning Strategy
       You are using {strategy} reasoning.
       Max iterations: {max_iterations ?? "unlimited"}
       Temperature: {temperature} (only if present)
       Thinking budget: {thinking_budget} tokens (only if present)
       ```
    3. Add internal helper `formatStepInstructions(stepName: string, step: Step): string` that formats:
       ```
       ## Current Step: {stepName}
       {step.description}

       {step.instructions}
       ```
       Handle missing description or instructions gracefully (omit blank lines).
    4. In compileStep:
       - Validate stepName exists in spec.steps, throw CompilerError if not
       - Call resolve(spec.steps) to get DAG levels, find this step's level
       - Build systemPromptSegment by concatenating formatStrategyPreamble + formatStepInstructions
       - Return CompiledStep with all fields populated (outputSchema from step, rest null/empty)

    All functions are pure. No I/O.
  </implementation>
</feature>
