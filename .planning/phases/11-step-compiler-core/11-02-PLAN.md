<feature>
  <name>compileStep: output schema formatting + retry policy + integration</name>
  <files>packages/core/compiler.ts, packages/core/compiler.test.ts</files>
  <behavior>
    Output schema compilation:

    Given a step with output_schema (e.g., gather_sources with its object schema):
    - systemPromptSegment includes "## Required Output Format" section
    - The section contains: "Your response must be valid JSON matching the following schema:"
    - Followed by the JSON schema rendered as a readable description
    - Includes instruction: "Ensure your output can be parsed as JSON. Include all required fields."
    - For function-calling compatibility: "If using structured output mode, this schema defines the response shape."
    - CompiledStep.outputSchema === step.output_schema (raw passthrough)

    Given a step WITHOUT output_schema (e.g., expand_search):
    - No "## Required Output Format" section in systemPromptSegment
    - CompiledStep.outputSchema === null

    Retry policy compilation:

    Given a step with retry config (e.g., gather_sources: max_attempts 3, initial_interval "1s", backoff_coefficient 2.0):
    - CompiledStep.retryPolicy === { maxAttempts: 3, initialInterval: "1s", backoffCoefficient: 2.0, maximumInterval: "1s", nonRetryableErrors: ["AuthenticationError", "RateLimitError"] }
    - Default values applied: maximumInterval defaults to initialInterval if not specified

    Given a step WITHOUT retry config:
    - CompiledStep.retryPolicy === null

    Given a step with partial retry config (only max_attempts):
    - Defaults: initialInterval "1s", backoffCoefficient 1.0, maximumInterval "60s", nonRetryableErrors []

    Integration (all 5 research-synthesizer steps):
    - gather_sources: dagLevel 0, has output_schema, has retry, has instructions
    - evaluate_credibility: dagLevel 1, no output_schema, no retry, has verification
    - synthesize: dagLevel 2, no output_schema, has branches
    - expand_search: dagLevel 3, no output_schema, minimal config
    - draft_report: dagLevel 3, no output_schema, has verification
  </behavior>
  <implementation>
    In compiler.ts, add to the existing implementation from Plan 01:

    1. Add internal helper `formatOutputSchema(schema: JsonSchemaObject): string` that formats:
       ```
       ## Required Output Format
       Your response must be valid JSON matching the following schema:

       ```json
       {JSON.stringify(schema, null, 2)}
       ```

       Ensure your output can be parsed as JSON. Include all required fields.
       If using structured output mode, this schema defines the response shape.
       ```
       This is model-agnostic: works for JSON mode (raw parsing) and function-calling (schema passthrough).

    2. Add internal helper `compileRetryPolicy(retry: RetryConfig): RetryPolicy` that maps:
       - max_attempts -> maxAttempts (default: 3)
       - initial_interval -> initialInterval (default: "1s")
       - backoff_coefficient -> backoffCoefficient (default: 1.0)
       - maximum_interval -> maximumInterval (default: "60s")
       - non_retryable_errors -> nonRetryableErrors (default: [])

    3. Update compileStep to:
       - Append formatOutputSchema section to systemPromptSegment when step.output_schema exists
       - Set outputSchema to step.output_schema or null
       - Set retryPolicy to compileRetryPolicy(step.retry) or null
       - qualityGates remains [] and selfReflection remains null (Phase 13)

    All functions pure. No I/O.
  </implementation>
</feature>
