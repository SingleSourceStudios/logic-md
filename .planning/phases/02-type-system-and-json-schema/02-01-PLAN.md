Create `packages/core/types.ts` containing every type from the LOGIC.md v1.0 specification. This is a single flat file (no src/ directory per project constraint).

Use the RESEARCH.md "Full Type Map from Spec" as the definitive reference. Implement ALL of the following types:

**String literal unions (not enums):**
- `ReasoningStrategy` = 'cot' | 'react' | 'tot' | 'got' | 'plan-execute' | 'custom'
- `Severity` = 'error' | 'warning' | 'info'
- `OnFailAction` = 'retry' | 'escalate' | 'skip' | 'abort' | 'revise'
- `ExecutionMode` = 'sequential' | 'parallel' | 'conditional'
- `JoinMode` = 'all' | 'any' | 'majority'
- `ValidationMode` = 'strict' | 'warn' | 'permissive'
- `ViolationAction` = 'reject' | 'coerce' | 'warn' | 'retry' | 'escalate'
- `FallbackStrategy` = 'graceful_degrade' | 'escalate' | 'abort' | 'retry_different'
- `SelfVerificationStrategy` = 'reflection' | 'rubric' | 'checklist' | 'critic'

**Common types:**
- `Expression` (string type alias for `{{ }}` expressions)
- `JsonSchemaObject` (loose interface with index signature for inline JSON Schema fragments)

**Interfaces (map every field from spec sections):**
- `LogicSpec` (root, section 2.1 — required: spec_version, name; optional: description, imports, reasoning, steps, contracts, quality_gates, decision_trees, fallback, global, nodes, edges, visual, metadata)
- `Import` (section 2.2 — required: ref, as)
- `Reasoning` (section 3 — required: strategy)
- `Step` (section 4.1 — all optional except described in spec)
- `ConfidenceConfig`, `Branch`, `RetryConfig`, `Verification` (step sub-types)
- `Contracts`, `ContractField`, `Capabilities`, `ContractValidation` (section 5)
- `QualityGates`, `Gate`, `Invariant`, `SelfVerification`, `ReflectionConfig`, `RubricConfig`, `RubricCriterion` (section 6)
- `DecisionTree`, `DecisionNode`, `DecisionBranch`, `Terminal` (section 7)
- `Fallback`, `EscalationLevel`, `DegradationRule` (section 8)
- `GlobalConfig`, `NodeRef`, `Edge` (section 9.1 workflow-level)
- `Visual`, `InspectorField`, `VisualPorts`, `VisualPort` (section 10)

**Required/optional rules (critical — follow spec exactly):**
- LogicSpec: `spec_version` and `name` required, everything else optional
- Import: `ref` and `as` both required
- Reasoning: `strategy` required
- Branch: `then` required; `if` and `default` optional
- Verification: `check` and `on_fail` required
- Gate: `name` and `check` required
- Invariant: `name` and `check` required
- DecisionTree: `root` and `nodes` required
- DecisionNode: `condition` and `branches` required
- DecisionBranch: `next` required
- Terminal: `action` required
- EscalationLevel: `level`, `trigger`, `action` required
- DegradationRule: `when`, `fallback_to` required
- Edge: `from` and `to` required
- ContractField: `name` and `type` required
- InspectorField: `key`, `label`, `type` required

Add JSDoc comments on the root `LogicSpec` interface and on each major section interface (Reasoning, Step, Contracts, etc.) referencing the spec section number.

Do NOT use TypeScript enums — use string literal union types only.
Do NOT use `any` — use `unknown` for truly open-ended types (metadata, strategy_config).
