  <action>
Create `packages/core/schema.json` — a complete JSON Schema draft-07 file that mirrors the TypeScript types from `types.ts`.

**Schema metadata:**
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://logic-md.dev/schema/v1.0/logic-spec.json",
  "title": "LogicSpec",
  "description": "LOGIC.md v1.0 specification schema"
}
```

**Root object:**
- `type: "object"`
- `required: ["spec_version", "name"]`
- `additionalProperties: false` (per research recommendation — strict validation, Phase 4 can downgrade to warnings)
- Properties: spec_version (const "1.0"), name (string, minLength 1), description, imports, reasoning, steps, contracts, quality_gates, decision_trees, fallback, global, nodes, edges, visual, metadata

**All definitions (use `$ref: "#/definitions/X"` pattern):**

Define EVERY sub-type as a definition. Key ones:

- `Import` — required: ref, as
- `Reasoning` — required: strategy; strategy is enum of the 6 values
- `Step` — all optional fields; uses $ref for sub-types
- `ConfidenceConfig`, `Branch`, `RetryConfig`, `Verification`
- `Contracts`, `ContractField`, `Capabilities`, `ContractValidation`
- `QualityGates`, `Gate`, `Invariant`, `SelfVerification`, `ReflectionConfig`, `RubricConfig`, `RubricCriterion`
- `DecisionTree`, `DecisionNode`, `DecisionBranch`, `Terminal`
- `Fallback`, `EscalationLevel`, `DegradationRule`
- `GlobalConfig`, `NodeRef`, `Edge`
- `Visual`, `InspectorField`, `VisualPorts`, `VisualPort`
- `JsonSchemaObject` — type object with additionalProperties true (user-authored inline schemas are open-ended)
- `Expression` — type string

**Critical details:**
- `steps` is `type: object` with `additionalProperties: { "$ref": "#/definitions/Step" }` (map<string, Step>)
- `decision_trees` is `type: object` with `additionalProperties: { "$ref": "#/definitions/DecisionTree" }`
- `nodes` is `type: object` with `additionalProperties: { "$ref": "#/definitions/NodeRef" }`
- `metadata` is `type: object` with `additionalProperties: true`
- String literal unions become `enum` arrays in JSON Schema
- Expression fields are just `type: string` (no pattern validation — Phase 5 handles parsing)
- Use `format: uri` where the spec mentions URLs (source_url in contracts)
- Do NOT use draft-2020-12 keywords (this is draft-07 for ajv default)

**Required arrays within definitions must match TypeScript required fields exactly.** Refer to Task 1 of Plan 01 for the required/optional field list.
